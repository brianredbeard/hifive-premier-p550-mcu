# Contributors and Licensing Information

This document outlines all copyright assignments, licenses, and contributors for
this STM32F407VET6 BMC (Baseboard Management Controller) firmware project, which
is based on STM32CubeMX and integrates multiple third-party components.

## Project Overview

This is an open-source embedded firmware project for a Baseboard Management
Controller (BMC) running on STM32F407VET6 microcontroller. The project
integrates code from multiple sources with different licenses, all of which are
compatible with open-source distribution.

## License Compatibility

This project combines code under the following licenses:

- **GPL-2.0-only** (Project-specific code)
- **MIT** (FreeRTOS)
- **BSD-3-Clause** (STM32 HAL Driver, LwIP)
- **Apache-2.0** (CMSIS, CMSIS-RTOS v2)

### Compatibility Analysis

**GPL-2.0-only is the governing license** for the combined work because:

1. **GPL-2.0-only is a copyleft license** that requires derivative works to be
   distributed under GPL-2.0

1. **All third-party licenses are GPL-compatible permissive licenses**:

   - MIT License: GPL-compatible ✓
   - BSD-3-Clause: GPL-compatible ✓
   - Apache-2.0: GPL-compatible ✓ (when combined with GPLv2, Apache-2.0 is
     generally compatible, though Apache-2.0 has additional patent grant
     provisions)

1. **Distribution Requirements**:

   - The entire firmware binary and source distribution must comply with
     GPL-2.0-only terms
   - Third-party components retain their original copyright notices
   - Modified third-party files must be marked as changed (GPL-2.0 requirement)
   - Source code must be made available when distributing binaries

### Important Notes for Open Source Distribution

1. **When distributing this firmware** (binary or source):

   - Include this CONTRIBUTORS.md file
   - Include full text of GPL-2.0 license
   - Include third-party license files from `Drivers/` and `Middlewares/`
     directories
   - Provide access to complete source code (GPL-2.0 requirement)
   - Maintain all copyright notices

1. **When modifying third-party code**:

   - Retain original copyright notices
   - Add modification notices indicating changes made
   - Respect original license terms while complying with GPL-2.0 for the
     combined work

1. **CubeMX Regeneration**:

   - STM32CubeMX-generated code can be regenerated and modified
   - Keep STM32 copyright notices in generated files
   - Mark any manual modifications to generated code

1. **Commercial Use**:

   - GPL-2.0-only permits commercial use
   - Binary distributions must include source code or written offer for source
   - All third-party licenses also permit commercial use
